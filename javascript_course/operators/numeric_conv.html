Numeric conversion using unary +

In JavaScript, the + operator comes in two forms: binary and unary.

The binary plus is used to add two values together. The unary plus (a single + before a value) attempts to convert that value into a number.

Example: Unary Plus on Numbers

let m = 5;
alert(+m); // 5

let n = -8;
alert(+n); // -8

Applying unary + to actual numbers does not change them.

Example: Unary Plus on Non-Number Values

alert(+false); // 0
alert(+"7");   // 7
alert(+"");    // 0

As shown, values like booleans and strings are converted into numbers.

The unary + behaves exactly like Number(...) but is shorter and often preferred for concise code.

Real-World Scenario: Getting Numeric Input from Forms

When data is collected from HTML form fields, the values are usually of type string, even if the user entered a number. Using binary + with strings results in string concatenation. 

let qty1 = "6";
let qty2 = "4";

alert(qty1 + qty2); // "64"

The result is a string combination, not a numerical sum.

Fixing the Problem Using Unary Plus

let qty1 = "6";
let qty2 = "4";

alert(+qty1 + +qty2); // 10

Here, both values are converted to numbers using unary + before being added.

Alternatively, you could use the longer version: 

alert(Number(qty1) + Number(qty2)); // 10
