Modify-in-place

In programming, itâ€™s common to update the value of a variable by applying an operation and saving the result back to that same variable. This pattern is called "modify-in-place."

Take the following example:

let total = 4;
total = total + 6;
total = total * 2;

Here, we first add 6 to total, then multiply the result by 2. This works fine, but JavaScript provides a shorthand syntax to make such operations cleaner and more concise.

Instead of writing the full expression again, we can use compound assignment operators: 

let total = 4;
total += 6; // total becomes 10 (same as total = total + 6)
total *= 2; // total becomes 20 (same as total = total * 2)

alert(total); // 20

These modify-and-assign operators are available for many arithmetic and bitwise operations, including:

  • += for addition
  • -= for subtraction
  • *= for multiplication
  • /= for division
  • %= for remainder
  • And others...

Operator Precedence Reminder

Just like the regular assignment =, these compound operators have low precedence. This means the right-hand side expression is fully evaluated first before applying the operation.

Example:

let value = 3;

value += 2 * 4; // right-hand side (2 * 4) is evaluated first â†’ value += 8

alert(value); // 11

This is the same as:

value = value + (2 * 4);
