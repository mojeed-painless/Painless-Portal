Assignment Operator

In JavaScript, the assignment operator (=) is not just a way to store values in variables â€” it is actually an operator with its own precedence level. It has a very low precedence of 2, meaning other operations are usually performed before it in an expression.

For example, consider the following expression:

let x = 3 * 3 + 2;

alert(x); // 11

The multiplication and addition are executed first due to their higher precedence, then the result is assigned to x.

Assignment Returns a Value

Unlike what we might assume, = is not a "special rule" in the language â€” itâ€™s just another operator. Like other operators, it returns a value.

This means the result of an assignment can be used directly in another expression. Letâ€™s see an example: 

let alpha = 5;
let beta = 7;

let result = 20 - (alpha = beta + 4);

alert(alpha); // 11
alert(result); // 9

Here, (alpha = beta + 4) returns 11, which is then used in the subtraction. So, 20 - 11 gives 9.

This behavior is sometimes used in library code, although it may reduce clarity. So, while it's important to understand, try to avoid using it in this way for regular programming unless necessary.

Chaining Assignments

Assignment operations can also be chained from right to left. This is useful when you want to assign the same value to multiple variables at once. 

let one, two, three;

one = two = three = 8 + 2;

alert(one);   // 10
alert(two);   // 10
alert(three); // 10

In this example, 8 + 2 is evaluated first. The result 10 is assigned to three, then to two, and finally to one.

Even though this chaining works, for better readability, it's often preferred to write assignments like this: 

let one, two, three;

three = 8 + 2;
two = three;
one = three;

This makes the code easier to understand and debug.
