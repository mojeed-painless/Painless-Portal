After successfully connecting your JavaScript file to your HTML document using the <script src="yourfile.js"> method, the next question is: “How do I know if it actually works?”

Great question! One simple and effective way to test your connection is to write your very first JavaScript code using a built-in function called alert(). Let's explore what it does.

Using the alert( ) Function

The alert() function is one of the most basic JavaScript tools used to display a pop-up message box in your browser. It pauses the page and shows a message to the user in a dialog box. It's often used for testing and quick notifications.

Example:

// add this in your script.js
alert("Hello, World! This means your JavaScript is working.");

Then in your HTML file:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title> My First JS Test </title>
</head>
<body>

    <h1> Testing JavaScript Connection </h1>
                    
    <script src="script.js"></script>

</body>
</html>

When you open this HTML file in your browser, you’ll immediately see a pop-up message saying: “Hello, World! This means your JavaScript is working.” This confirms that your script is properly connected and executed.

Note: The alert box pauses your script and stops the user from interacting with the page until it's dismissed.

That’s why it’s mostly used for testing and debugging during development, not in production websites.

Introducing the Developer Console

Once you're comfortable with alert(), the next step is using the browser console — a powerful tool for developers to test and debug JavaScript code without disturbing the page or user experience.

You can write messages or output results using the console.log() function. It doesn’t show anything on the webpage, but instead prints the result to your browser’s developer tools console.

Example:

console.log("Hello from the console!");

To see this message, you need to open the Developer Console in your browser.

How to open the Console

  • Windows/Linux: Press Ctrl + Shift + I or F12
  • Mac: Press Cmd + Option + I
  • Then, click the Console tab in the Developer Tools pane.

You should see the message printed like this:

Hello from the console!

Why use the Console?

Developers rely on the console to:

  • Test JavaScript snippets quickly
  • Check values of variables
  • Track errors and debug issues
  • View outputs without disrupting the web page

Practical Example

Here’s a simple interactive page that combines both alert() and console.log(): 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title> Hello World Example </title>
</head>
<body>

    <h2> Click the button to test your script </h2>
    <button onclick="sayHello()"> Click Me </button>

    <script src="script.js"></script>

</body>
</html>

And in script.js add:

function sayHello() {
    alert("Hello, World!");
    console.log("The button was clicked and alert was shown.");
}

When you click the button, it will show an alert with "Hello, World!" and log a message in the console.

Don't worry about the function sayHello() part for now, you will learn more about it later on.

What you’ve Learned

  • What the browser console is and how to use console.log()
  • Keyboard shortcuts to access Developer Tools
