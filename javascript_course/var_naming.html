Naming Variables

There are two main rules for naming variables in JavaScript:

  • The name must contain only letters, digits, or the symbols $ and _.
  • The first character must not be a digit.

Examples of valid names:

let user;
let product456;

When variable names contain multiple words, the camelCase convention is often used. This means the first word starts lowercase, and each subsequent word starts with a capital letter: 

let customerPhoneNumber;

You can also use $ and _ as part of variable names. They don‚Äôt have any special meaning: 

let $value = 5;
let _tempCount = 10;

alert($value + _tempCount); // 15

Examples of Incorrect Variable Names

let 7score;   // Cannot start with a digit
let user-name;  // Hyphens aren't allowed

Case Sensitivity

JavaScript variable names are case-sensitive. That means:

let dog = "woof";
let Dog = "bark";

console.log(dog); // "woof"
console.log(Dog); // "bark"

Reserved Words

Some words are reserved by JavaScript and cannot be used as variable names, such as let, return, function, and class. 

let return = 10; // Error: "return" is a reserved keyword

Naming Things the Right Way

Variable names should be clear and descriptive. This makes code easier to read, understand, and maintain. For example: 

// Good
let currentUser;
let orderTotal;

// Bad
let a;
let data;
let val;

Some good naming practices include:

  • Use meaningful names like cartItemCount instead of c.
  • Be consistent with naming conventions in your team or project.
  • Avoid vague names like thing, stuff, or value.

Conclusion

Naming variables properly is a key skill in JavaScript. It helps your code stay clean, understandable, and bug-free. Always use descriptive, readable, and consistent names.
