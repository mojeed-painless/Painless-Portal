Browser User Interaction

In web browsers, JavaScript allows developers to interact with users through built-in dialog boxes. The three main functions used for interaction are:

  • alert()
  • prompt()
  • confirm()

alert( )

The alert function displays a pop-up message and waits until the user clicks ‚ÄúOK‚Äù. 

alert("Welcome to our site!");

This method creates a modal dialog ‚Äî meaning it pauses the entire page until the message is dismissed.

prompt( )

The prompt function displays a dialog with a message and an input field for the user to type something.

Syntax:

prompt(message, [defaultInput]);
  • message: What will be displayed to the user. Usually a message requesting for user's input
  • defaultInput: This is the default value that will appear in the input field, then the users can cancel it and put their desired value.

Example

prompt("What's your name?", "Guest");

If the user clicks OK, the typed value is returned. If Cancel is clicked or ESC is pressed, null is returned.

The defaultInput can be left blank using an empty string

Example

prompt("What's your name?", "");

Now users would not see any default value in the input field, but can still input their desired value

Storing User Input

It should be noted that we can also store the user input in a variable and use them when ever we need to

Example:

let userName = prompt("Enter your username:", "user123");

alert(`Hello, ${userName} !`);

confirm( )

This function presents a message with OK and Cancel buttons, usually to confirm an action.

Example:

confirm("Do you want to continue?");

Returns true if OK is clicked, and false if Cancel is clicked.

Example:

let registeredUser = confirm("Are you a registered user?");
alert(registeredUser) // true if OK is clicked

Summary

  • alert: shows a message box with only an OK button.
  • prompt: asks the user for input and returns the input text or null.
  • confirm: asks a yes/no question and returns true or false based on the user's choice.

Note: These dialog boxes are basic and cannot be styled. Their appearance and position depend on the user's browser.
