The "typeof" operator

The typeof operator helps determine the data type of a given value. It's particularly useful when working with values of various types or when you just want to quickly check what kind of data you're dealing with.

When you run typeof someValue, it returns a string indicating the valueâ€™s type: 

typeof undefined       // "undefined"
typeof 42              // "number"
typeof 900719925n      // "bigint"
typeof false           // "boolean"
typeof "hello"         // "string"
typeof Symbol("key")   // "symbol"
typeof null            // "object"

Note:

  • Why typeof null returns "object": This is a long-standing bug in JavaScript. Although null is a special value with its own type, typeof null mistakenly returns "object". It has been preserved for backward compatibility.

typeof(value) vs typeof value

Both of the following are valid and behave the same: 

typeof userName
typeof(userName)

While the second version uses parentheses, they are purely for grouping, just like in mathematical expressions. Some developers prefer this style, but the version without parentheses is more commonly used.
