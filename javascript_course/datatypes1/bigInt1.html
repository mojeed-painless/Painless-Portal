BigInt

In JavaScript, the number type cannot safely represent integer values larger than 253 - 1 (which is 9007199254740991) or less than -9007199254740991. These limits define the safe integer range.

Problem

Although JavaScript can technically store larger integers, doing so may lead to precision errors. For example: 

console.log(9007199254740991 + 1); // 9007199254740992
console.log(9007199254740991 + 2); // 9007199254740992 (wrong!)

Both expressions return the same result, even though they should be different. This happens because JavaScript cannot accurately distinguish numbers outside the safe range.

Solution

To solve this, JavaScript introduced the BigInt type, which allows you to work with integers of arbitrary length.

A BigInt is created by adding an n at the end of an integer literal: 

// the "n" at the end means it's a BigInt
const accountBalance = 987654321987654321987654321987654321n;

BigInts are rarely needed for everyday tasks, but they are very useful in fields like cryptography or handling precise timestamps.
