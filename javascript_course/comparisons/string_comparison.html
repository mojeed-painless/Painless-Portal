String comparison

In JavaScript, comparing strings follows the lexicographical (dictionary-like) order, meaning each character is compared one by one based on their Unicode values.

Rules:

This method checks the characters from left to right:

  • It starts by comparing the first characters of each string.
  • If the first characters differ, the result is decided based on their Unicode values.
  • If they are the same, it continues to the next pair of characters.
  • This continues until a difference is found or until one string ends.
  • If one string ends and all characters matched, the longer string is considered greater.

Examples:

alert('Apple' > 'Ape');     // true: 'p' > 'e'
alert('Drip' > 'Drop');     // true: 'i' > 'o'
alert('Ant' > 'Antelope');  // false: shorter string is smaller

Note:

This comparison is based on Unicode values, not a real dictionary. That means case sensitivity is important. Uppercase letters come before lowercase letters in Unicode.

Example with Case Sensitivity:

alert('bat' > 'Ball');   // true: lowercase 'b' has higher Unicode value than uppercase 'B'
alert('Zoo' > 'zoo');    // false: 'Z' is less than 'z'

JavaScript uses Unicode encoding, so each character has a numeric position. The higher the position, the greater it is when compared.
