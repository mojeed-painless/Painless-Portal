Comparison of different types

In JavaScript, when values of different types are compared using comparison operators (>, <, ==, etc.), the language automatically converts them into a common type, typically a number, before performing the comparison.

JavaScript also converts Boolean values (true and false) into numbers when necessary:

An Interesting Case of Comparison

Sometimes, JavaScript will treat two values as equal, even if they behave differently in Boolean logic. 

let num = 0;
alert(Boolean(num)); // false

let text = "0";
alert(Boolean(text)); // true

alert(num == text); // true!

Why This Happens:

When using ==, JavaScript converts both values to numbers: 0 == "0" becomes 0 == 0, which is true.

But when explicitly converting to Boolean:

  • 0 is falsy
  • "0" (a non-empty string) is truthy

This can lead to unexpected behavior if you're not aware of the type coercion rules.
